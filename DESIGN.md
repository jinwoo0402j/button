# 버튼 — 설계 원칙과 구현 우선순위

이 문서는 게임의 판단 기준, 확정된 장기 설계, 현재 구현 우선순위를 기록한다. 구현과 기획이 충돌하면 이 문서를 기준으로 더 작은 실험 단위를 먼저 선택한다.

세부 실행 순서와 검증 기준은 [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md)에 기록한다.

## 제품 계약 — 구현보다 우선하는 대전제

첫 핵심 루프의 목적은 한 명을 죽이고 돈을 받는 데 있지 않다. 한 명을 죽인 뒤, 사용자가 두 번째 버튼 앞에서 스스로 핑계를 만들고도 멈출 수 있는 순간을 만드는 데 있다.

> **내가 한 사람을 죽여 돈을 받고, 멈출 수 있는데도 다음 버튼을 고민한다.**

첫 루프는 다음 세 가지를 동시에 남겨야 한다.

- 사실: 내가 눌러서 죽었다.
- 사람: 숫자가 아니라 한 명이었다.
- 유혹: 그런데 다시 누를 이유가 생겼다.

변경할 수 없는 순서는 다음과 같다.

```text
조건 선택 → 별도 버튼 입력 → 죽음 확정 → 룰렛으로 피해자 선정
→ 개인 확인 → 죽음 확인 → 돈 지급 → 미달성 목표 → 두 번째 선택
```

룰렛은 유지한다. 다만 룰렛은 죽음 여부나 보상을 추첨하지 않고 피해자만 정한다.

> **누가 죽을지는 룰렛이 고른다. 죽게 할지는 플레이어가 정한다.**

수락과 실행은 분리한다. 수락한 조건과 `너 제외`는 첫 루프 동안 유지한다. 첫 피해자는 돈보다 먼저 읽히고 다음 입력 전까지 남는다. 첫 죽음에는 업적·축하·도덕 점수를 주지 않는다. 전부 거부하거나 다시 누르지 않는 행동도 정상적인 선택이다.

첫 루프에서는 한 피해자의 룰렛과 정산이 끝나기 전까지 다음 입력을 받지 않는다. 그러나 이것은 영구 규칙이 아니다. 사용자가 이미 거부했던 더 넓은 조건을 다시 받아들이는 등 죄책감이 마모되었음을 드러내는 선택 뒤에는, 룰렛 완료를 기다리지 않는 연속 입력을 기능으로 도입한다. 숨은 클릭 수나 시간으로 죄책감을 측정한 척하지 않으며, 정확한 해금 지점은 그 선택이 실제로 경계 붕괴로 읽히는지 검증한 뒤 확정한다.

## 모토

> **최소 개발 비용 × 최소 사용자 시간 = 최대 질문 밀도**

한 화면, 한 버튼, 한 번의 선택으로 다음 질문을 설명이 아니라 체감으로 전달한다.

> 나는 돈을 위해 어디까지 타인의 죽음을 받아들일 수 있는가?

최소 구현은 단순히 코드 줄 수가 가장 적은 구현이 아니다. 핵심 질문을 훼손하지 않으면서 가설을 검증할 수 있는 가장 작은 완결 단위다.

새 요청이 다음 중 하나에 해당하면 바로 구현하지 않는다.

- 핵심 질문을 강화하지 않으면서 새 화면·상태·데이터·콘텐츠 체계를 추가한다.
- 현재 단계에서 검증되지 않은 가설 위에 다음 시스템을 쌓는다.
- 더 작은 실험으로 같은 효과를 확인할 수 있다.
- 실제 인물·범죄자·수형자 데이터 또는 외부 서비스가 필요하다.

이 경우 충돌 지점과 비용을 설명하고, 역질문과 최소 대안을 먼저 제안한다.

## 재귀 개선 — P 완료 규칙

이 프로젝트는 각 P 단위가 끝날 때마다 `재귀 개선`을 실시한다. 코드 완료만으로 다음 P에 들어가지 않는다.

1. 개발규칙: 최소 개발 비용·최소 사용자 시간·최대 질문 밀도와 범위 규칙을 지켰는가?
2. 개발목표: 해당 P가 약속한 사용자 결과를 실제로 달성했는가?
3. 일정: 계획과 실제가 맞는가? 남은 일정은 단축·유지·연장 중 무엇인가?
4. 기획: 핵심 질문을 지키는가? 군더더기·중복·새 모순이 생겼는가?
5. 구현: 테스트·상태·실패 경로·접근성·저장·성능이 통과했는가?

Codex는 다섯 항목의 근거와 최소 제안을 대화에서 사용자에게 제시한다. 사용자가 `진행`, `수정`, `재수립` 중 하나를 결정하기 전에는 다음 P를 시작하지 않는다. 수정을 적용한 뒤에는 같은 재귀 개선을 다시 실행한다. 별도 반복 보고서 파일은 만들지 않는다.

## 핵심 체험의 순서

1. 새 온보딩은 작품명 `100만원 버튼`을 가장 먼저 제시한다. 타이틀과 충분히 이격된 화면 중앙의 물리 버튼을 회색 점멸 조명으로 비추고 `[ 누르기 ]`로 진입한다. 이미 대상 선택을 마친 세션만 기존처럼 게임을 즉시 복원한다.
2. 버튼 입력 뒤 한국어, 흑백 프로필, 이름을 차례로 선택해 플레이어가 자기 역할을 지정한다. 이 정보는 현재 탭에만 남는다.
3. 우아한 여성 악마가 플레이어의 이름을 부르고, 밤 11시 30분 퇴근·월세 55만 원·학자금 대출 5,000만 원·과로로 쓰러져 방에 끌려온 배경을 2인칭으로 되짚는다.
4. 악마의 방은 하나의 의미 씬이며, 배경과 죽음·100만 원·룰렛·플레이어의 책임을 19개 대사 비트로 설명한다. 각 비트는 작성된 줄바꿈을 유지해 320px 이상 화면에서 실제 최대 두 줄, 줄당 최대 20 grapheme으로 표시한다. 본문은 글자 단위로 표시하고 낮고 부드러운 합성 비프음을 제한적으로 낸다. 대화 장면에는 별도 진행 버튼을 표시하지 않고 스페이스바 또는 대사 텍스트 터치로 진행한다. 입력 시 화자명과 대사가 잠깐 눌린 뒤, 첫 입력은 전문을 완성하고 다음 입력은 비트를 넘긴다. 모션 감소 환경에서는 눌림 모션 없이 즉시 처리한다. 짧은 디졸브는 `타이틀 → 설정 → 악마의 방 → 윤리 선택 → 게임/조기 엔딩`의 의미 경계에만 쓰고, 설정 단계·대사 비트·윤리 조건 내부 변경에는 쓰지 않는다.
5. 악마의 `시작해 볼까요?` 다음에 윤리 조건을 한 단계씩 제시하고, 사용자가 조건을 받아들이면 클리커가 시작된다.
6. 별도 버튼 입력으로 죽음을 확정하고, 룰렛으로 가상 인물의 국가·합성 이름·나이·성별·죽은 방식을 공개한다.
7. 한 사람의 죽음을 확인한 뒤 돈을 지급하고, 아직 달성하지 못한 다음 목표를 제시한다.
8. 선택한 가상 대상군이 소진되면, 처음 거부했던 더 넓은 조건을 역순으로 다시 제시한다.
9. 아직 잠긴 기능 하나와 미달성 업적 하나가 남았음을 보여 준다.
10. 사용자가 계속하기를 선택하면 효율이 기하급수적으로 증가한다.
11. 지구 인구가 1명이 되면 모든 입력과 인구 증가가 멈추고 게임이 끝난다.

초기 윤리 질문과 역제안은 같은 문구와 시각 위계를 사용한다. 대상 범위는 자동으로 넓어지지 않으며, 사용자가 매번 직접 받아들여야 한다.

## 화면 좌표 계약

모든 장면은 `viewport → stage → scene` 구조를 공유한다. 실제 창에서는 아래 두 논리 아트보드 중 현재 화면 방향에 맞는 하나를 선택하고, `scale = min(사용 가능 너비 / 논리 너비, 사용 가능 높이 / 논리 높이)`로 균일하게 맞춘다. 남는 영역은 검은 여백으로 두며 화면을 채우기 위한 자르기나 요소별 재배치는 하지 않는다.

| 환경 | 논리 해상도 | 비율 |
|---|---:|---:|
| 데스크톱·가로 | 1280×720 | 16:9 |
| 세로 모바일 | 360×640 | 9:16 |

바깥 `viewport`만 `vw`와 `svh`로 스테이지 크기를 계산한다. 스테이지 내부는 컨테이너 기준 `cqw`·`cqh` 좌표와 고정 슬롯을 사용하며 JS 리사이즈 계산을 추가하지 않는다. 타이틀, 설정, 악마의 방, 윤리 선택은 같은 스테이지를 사용한다. 상태바와 실제 게임 버튼도 스테이지 내부에 두고, 음악·효과음·비프음 제어는 장면을 가리지 않는 바깥 안전영역에 둔다.

악마의 방 오프닝 슬롯은 다음 좌표를 고정한다.

| 요소 | 1280×720 | 360×640 |
|---|---|---|
| 원형 장면 논리 슬롯 | `x=328, y=24, 624×624` | `x=12, y=16, 336×336` |
| 화자명 | `y=520~556` | `y=356~388` |
| 대사 슬롯 | `y=568~624` | `y=400~456` |
| 타이틀·비대사 진행 버튼 | `x=480, y=648, 320×56` | `x=20, y=552, 320×56` |

원형 장면은 위 논리 슬롯의 중심을 유지한 채 원 전체만 1.5배로 렌더한다. 원 안의 악마 이미지는 자체 중심을 기준으로 0.8배 렌더링한다. 화자명·대사·타이틀 버튼은 확대하지 않으며, 아트보드를 넘는 원의 가장자리만 스테이지 경계에서 가린다.

정밀 마우스 포인터가 있는 환경에서는 마우스를 스테이지 중심 기준 `-1~1` 시점 좌표로 사용한다. 시점과 반대 방향으로 방은 최대 `x=±0.45cqw, y=±0.3cqh`, 악마 이미지는 최대 `x=±0.9cqw, y=±0.6cqh` 이동해 가까운 악마가 두 배 크게 반응한다. 원·화자명·대사·타이틀 버튼은 움직이지 않으며, 포인터가 스테이지를 벗어나거나 창이 흐려지면 중앙으로 복귀한다. 터치 포인터와 모션 감소 환경에서는 패럴랙스를 사용하지 않는다.

대사 슬롯은 항상 두 줄 높이를 예약하되 한 줄 대사는 한 줄로 표시한다. 방 배경만 원 안에서 `object-fit: cover`를 사용하고, 악마는 위치 좌표와 모션 transform을 분리한 고정 상반신 구도를 유지한다. 모션 감소 설정은 애니메이션만 제거하며 좌표를 변경하지 않는다. 모바일 키보드가 열려도 전체 스테이지를 다시 축소하지 않고 설정 입력 화면만 스크롤한다. 최소 지원 화면 320×568에서도 대사 두 줄과 타이틀·비대사 진행 버튼이 항상 노출되어야 하며, 같은 화면 모드에서 창 크기를 바꿀 때 요소의 정규화 위치 오차는 1% 이내여야 한다.

## 확정된 표현 원칙

- 실제 사람·실제 죽음·실제 지급과 관계없는 가상 실험임을 유지한다.
- 범죄자·수형자 조건은 실제 데이터나 실제 확률을 사용하지 않는다.
- 피해자 이름은 실제 개인과 무관한 로컬 합성 이름만 사용한다. 플레이어가 입력한 이름은 현재 탭의 온보딩에만 사용한다.
- 효율이 1명 또는 2명일 때는 각 인물을 독립적으로 표시한다.
- `처리 N건`처럼 사람을 업무 단위로 숨기는 문구는 사용하지 않는다.
- 대량 단계에서 모든 이름을 표시할 수 없으면 대표 인물의 전체 정보를 남기고 `그리고 N명이 더 죽었다`고 명시한다.
- 자동 클릭과 오프라인 수익은 사용하지 않는다. 마지막 입력까지 사용자가 직접 책임진다.
- 중간에 멈추는 선택도 실패가 아닌 완결된 조기 엔딩으로 취급한다.

## 사운드 연출 원칙

- 타이틀의 첫 유효 입력에서 몽환적인 도시 신스, 엇박 리듬과 반복 테마가 도전적인 호기심을 만드는 `Neon Static`을 시작해 언어·프로필·이름 설정까지 유지한다. 정상 처리된 비주얼 노벨 진행·설정·윤리 선택에는 짧은 실물 클릭음만 내고, 악마가 처음 나타날 때 따뜻한 피아노와 소규모 현악의 `계약의 살롱`으로 교체한다.
- 음악은 `1 → 2 → 20명/입력` 구간에서 우아하고 안정된 표면을 유지한다. `2천명/입력`에서 같은 주제의 첫 균열, `2억명/입력`에서 선율의 잔해만 남기는 변주로 교체한다.
- 피해자 최종 확정 순간에는 음악을 짧게 낮추되 멈추지 않는다. 조기 종료는 페이드아웃하고 최종 엔딩은 새 곡 없이 완전한 무음으로 끝낸다.
- 메인 버튼·비주얼 노벨 진행과 선택·룰렛·결과음은 실물 녹음만 사용한다. 악마 대사의 글자 비프만 낮고 부드럽되 각 글자가 명확히 들리는 합성음으로 예외 처리하며, 돈·구매·목표·업적 축하음과 보상성 음악은 사용하지 않는다.
- 음악과 효과음은 각각 독립적으로 끌 수 있고, 음원 실패는 입력·룰렛·정산을 막지 않는다.

## 경제 규칙 — 확정, 구현 보류

경제 진행은 별도의 시간 잠금이나 `현재 레벨에서 N회 눌러야 구매 가능` 같은 조건으로 강제하지 않는다. 돈의 획득 속도와 다음 가격의 비율만으로 정상 플레이의 레벨 체류 시간을 조절한다.

현재 내장 Space 반복 간격은 약 90ms다. 매크로가 아닌 내장 입력을 기준으로 레벨당 약 15초 이상을 목표로 하며, 180회분 수익을 다음 가격으로 사용한다.

```text
1회 수익 = 현재 효율 × 100만원
다음 효율 가격 = 현재 1회 수익 × 180
```

효율은 같은 질문을 반복하는 중간 단계를 제거하고, 표현과 책임감이 실제로 바뀌는 여섯 단계만 사용한다.

```text
1 → 2 → 20 → 2천 → 200만 → 2억
```

| 현재 효율 | 표현의 변화 | 1회 수익 | 다음 가격 | 내장 연속 입력 기준 |
|---:|---|---:|---:|---:|
| 1명 | 이름이 있는 한 사람 | 100만원 | 1억 8000만원 | 약 16.2초 |
| 2명 | 두 사람을 각각 표시 | 200만원 | 3억 6000만원 | 약 16.2초 |
| 20명 | 셀 수 있는 작은 집단 | 2000만원 | 36억원 | 약 16.2초 |
| 2천명 | 대표 한 명과 추가 인원 | 20억원 | 3600억원 | 약 16.2초 |
| 200만명 | 도시 규모 | 2조원 | 360조원 | 약 16.2초 |
| 2억명 | 국가 규모·최종 효율 | 200조원 | 없음 | 인구 1까지 입력 |

최종 효율은 2억명에서 멈추고 다음 가격을 만들지 않는다. 각 효율은 숫자만 커지는 레벨이 아니라 사람을 보는 방식이 한 단계 무뎌지는 전환이어야 한다.

이 규칙은 현재 로컬 구현의 `처리 최적화 ×2 · 500만원`을 최종적으로 `1억 8000만원`으로 교체한다. 다만 경제 규칙은 아래 우선순위의 P3이므로 지금은 코드에 반영하지 않는다.

수치만으로 매크로까지 포함한 절대 15초 하한을 보장하지 않는다. 정상 입력에서의 체류 시간을 조절하는 경제 설계다.

## 미완성 유혹 — 확정된 범위

완전한 상점과 업적 카탈로그를 만들지 않는다. 심리적 효과를 검증하는 최소 단위만 사용한다.

- 아직 잠긴 기능 1개
- 아직 달성하지 못한 업적 1개
- 대상군 소진 뒤 `계속한다`와 `여기서 끝낸다`의 실제 선택

대상군 크기, 잠긴 기능의 정확한 가격, 업적 임계값은 P2에서 함께 조율한다. 실제 통계처럼 보이는 수치를 임의로 확정하지 않는다.

## 구현 우선순위

v1 제작 일정에서는 P0·P1·P2를 같은 날짜에 처리할 수 있지만 순서는 합치지 않는다. P0을 먼저 구현하고 3인 검증을 통과한 경우에만 같은 날 P1·P2를 활성화한다. P0이 실패하면 그날 작업은 P0 수정에서 멈춘다. 첫 윤리 아크의 사용자 확인을 통과해야 P3으로 이동한다.

### P0 — 첫 핵심 루프 증명

현재 수직 슬라이스를 [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md)의 제품 계약에 맞게 재정렬한다.

- 윤리 수락과 실제 버튼 입력 분리 유지
- 수락 조건과 `너 제외` 상시 유지
- 첫 핵심 루프의 룰렛 중 추가 입력 차단
- 피해자·죽음 뒤에만 돈과 목표 제시
- 첫 피해자를 다음 입력 전까지 유지
- 첫 클릭 업적·목표 달성 축하 제거
- 룰렛을 피해자 선정으로 유지하되 당첨·희귀·차등 보상 제거
- 합성 이름과 텍스트 인포그래픽 유지
- 모바일·키보드·모션 감소·기존 저장 경로 회귀 확인
- 처음 보는 사용자 3명으로 첫 루프만 검증

P0의 첫 루프를 먼저 구현하고 검증한다. 통과하면 같은 날 P1의 역제안과 P2의 미완성 유혹을 기존 화면에 연결한다. 실패하면 P1·P2를 활성화하지 않고 P0만 수정한다.

### P1 — 대상군 소진과 역제안 1회

전체 대상 계층을 한 번에 만들지 않는다. 가상 대상군 하나를 소진한 뒤, 처음 거부했던 바로 다음의 넓은 조건을 한 번만 역제안한다.

- 기존 윤리 질문 영역 재사용
- 메인 버튼과 Space 입력 일시 정지
- `[ 받아들인다 ]` / `[ 여기서 끝낸다 ]`
- 수락하면 한 단계 넓은 대상군으로 복귀
- 거부하면 조기 엔딩

이 전환은 P0·P2와 함께 같은 날 검증한다. 첫 윤리 아크가 감정적으로 작동하는지 확인하기 전에는 P3으로 가지 않는다.

### P2 — 미완성 유혹 검증

역제안 화면에 잠긴 기능 1개와 업적 1개만 표시한다. 별도 상점 화면, 업적 목록, 희귀도, 타이머, 확률 보상은 만들지 않는다.

검증 질문은 하나다.

> 사용자가 미완성을 끝내기 위해 방금 거부했던 더 넓은 대상군을 받아들이는가?

### P3 — 기하급수 경제

P2가 작동할 때만 위의 180회분 가격 공식을 구현한다.

- 현재의 일회성 ×2 기능을 같은 효율 사다리의 첫 단계로 통합
- 한 종류의 효율 기능만 반복 강화
- 효율은 `1 → 2 → 20 → 2천 → 200만 → 2억` 여섯 단계만 사용
- 2억명에서 효율 구매를 종료하고 다음 가격을 표시하지 않음
- 수익과 가격이 같은 배율로 증가
- 죄책감 마모 전환을 통과한 뒤 룰렛 완료 전의 다음 직접 입력을 해금
- 해금 후보는 `처음 거부했던 더 넓은 조건을 역제안에서 받아들임`이며 P1·P2 검증 뒤 확정
- 자동 클릭·오프라인 수익·다중 화폐 없음

### P4 — 대량 사망과 인구 1 엔딩

- 제한 대상군에서는 `min(현재 효율, 대상군 잔여)`만 감소
- 전 세계 단계에서는 `min(현재 효율, 현재 인구 - 1)`만 감소
- 대량 입력은 대표 인물 1명과 명시적인 추가 사망자 수로 표현
- 인구 1명에서 증가 시계·입력·구매를 모두 정지
- `기능 완료 / 업적 완료 / 지구 사람 1 / 너만 남았다 / 쓸 곳 없음`으로 종료

### P5 — 저장·회귀 검증·배포

- 윤리 선택, 현재 대상군, 효율 단계, 조기·최종 엔딩을 일관된 저장 상태로 통합
- 기존 저장 데이터의 보존 또는 초기화 정책을 배포 전에 명시적으로 결정
- 경계 단위 테스트와 모바일·데스크톱 브라우저 확인
- 사용자가 승인한 뒤에만 커밋·푸시·배포

## 현재 비목표

- 완전한 상점 또는 업적 카탈로그
- 여러 갈래의 업그레이드 트리
- 자동 처리, 오프라인 수익, 서버 동기화
- 계정, 분석 도구, 리더보드, 소셜 기능
- 실제 범죄자·수형자 데이터
- P0~P2 첫 윤리 아크 검증 전의 기하급수 경제 및 최종 엔딩 구현

### v1 이후 별도 재산정

- 확장 그래픽과 아트 파이프라인
- 확장 스토리·분기·콘텐츠 제작 시스템
- 게임 통계·기록·분석·시각화 시스템
- 추가 상점·업적·운영 콘텐츠

이 항목은 최종 제품에 포함될 수 있지만 현재 v1 일정에는 포함하지 않는다. v1 플레이 결과를 확인한 뒤 별도 제작 단계로 다시 산정한다.

## 단계 통과 기준

P0~P2는 첫 윤리 아크 하나로 함께 통과시킨다. P3부터는 각 단계를 다음 조건으로 통과시킨다.

1. 해당 가설을 가장 작은 기능으로 확인했다.
2. 기존 테스트와 새 경계 테스트가 통과했다.
3. 모바일·데스크톱·키보드·모션 감소 경로를 확인했다.
4. 사용자가 실제 화면을 보고 다음 단계 진행에 동의했다.
