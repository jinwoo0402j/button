## 산정 전제

- 1인 개발과 Codex 협업
- 평일 기준, 하루 4~6시간의 집중 구현·검증
- 현재 정적 HTML/CSS/JavaScript 구조 유지
- 백엔드, 계정, 실제 범죄자 데이터, 새 아트·음성 제작 없음
- 현재 범위인 P0~P5 고정
- 사용자 결정은 각 게이트에서 24시간 안에 이루어짐
- 첫 루프 검증 참가자 3명을 2026-07-17까지 확보

가용 시간이 하루 4시간보다 적거나 검토가 하루 늦어지면 이후 날짜도 같은 만큼 밀린다.

## 2026-07-13 작업 범위

내일의 목표는 P0 전체가 아니라 **첫 입력 경계가 깨지지 않는 기준선**이다.

1. 기준선 커밋 `f6f5e39`와 현재 31개 테스트의 의미를 분류한다.
2. `ethics / ready / rolling / refused`의 최소 상호작용 단계를 둔다.
3. 첫 룰렛 중 더블클릭·Space 홀드·혼합 입력을 차단한다.
4. 한 입력이 피해자 한 명과 정산 한 번만 만들도록 테스트한다.
5. 시간이 남으면 돈·인구·목표 정산을 룰렛 완료 뒤로 옮기는 골격까지만 만든다.

내일 완료 기준:

- 첫 룰렛이 끝나기 전 사망자는 정확히 한 명
- 윤리 수락 Space가 메인 버튼 입력으로 이어지지 않음
- 기존 자동 테스트와 새 입력 경계 테스트 통과
- P1 이후 기능을 건드리지 않음

## 사용자 시간 예산

사용자가 직접 확인해야 하는 지점은 다섯 번으로 제한한다.

| 예상일 | 결정 | 예상 시간 |
|---|---|---:|
| 7/21 | 첫 루프가 질문을 전달하는가 | 15분 |
| 7/30 | 역제안과 미완성 유혹이 작동하는가 | 15분 |
| 8/6 | 경제와 연속 입력 전환이 맞는가 | 20분 |
| 8/12 | 엔딩이 의도대로 남는가 | 20분 |
| 8/20 | v1을 출시할 것인가 | 15분 |

예상 총 사용자 검토 시간은 **약 85분**이다. 검토할 때는 여러 결정을 묶지 않고 해당 단계의 질문 하나만 제시한다.

## 크리티컬 패스

```text
P0 첫 루프 검증
→ P1 역제안
→ P2 미완성 유혹
→ P3 범용 효율·연속 입력
→ P4 대량 처리·엔딩
→ P5 저장 통합·출시 QA
```

- P0~P2는 감정적 가설이 앞 단계에 의존하므로 병렬 구현하지 않는다.
- P3는 P4가 사용할 `사망 수 + 대표 피해자` 구조까지 고려해 만들며, 대량 배열을 생성하지 않는다.
- 저장 스키마는 모든 상태가 확정된 P5에서 한 번만 올린다.

## 일정 위험과 자동 재산정 규칙

| 위험 | 영향 | 대응 |
|---|---:|---|
| P0 3인 검증 실패 | +2~5 작업일 | 실패한 인과·사람·유혹 층만 수정하고 신규 검증 |
| 검증 참가자 확보 지연 | 지연 일수만큼 | 7/13부터 3명을 먼저 확보 |
| P1·P2 수치가 감정을 만들지 못함 | +2~5 작업일 | 기능 추가 없이 대상군·가격·문구만 재조정 |
| 연속 입력의 중복 정산 | +1~3 작업일 | 입력 ID와 단일 정산 불변식부터 테스트 |
| 대량 단계에서 인물 배열 생성 | +2~4 작업일 | 대표 피해자 1명과 사망 수만 저장 |
| 저장 정책 결정 지연 | +2~3 작업일 | P4 종료 때 보존 또는 초기화를 결정 |
| 모바일·키보드·모션 감소 실패 | +1~2 작업일 | 각 단계에서 회귀 확인, P5에 몰지 않음 |
| 새 상점·업적·분기 추가 | 전체 재산정 | 현재 단계의 질문과 직접 관련 없으면 보류 |
| 다른 작업이 같은 코드 파일을 동시에 수정 | +1~2 작업일 | `f6f5e39`를 기준선으로 삼고 코드 편집 세션을 겹치지 않음 |

한 단계가 실패하면 이후 날짜를 유지한 채 기능을 밀어 넣지 않는다. 실패 기간만큼 출시 예상일을 뒤로 이동하고 이 문서를 갱신한다.

## 출시 차단 조건

- 플레이어 입력과 죽음의 인과가 불명확함
- 룰렛이 죽음 여부나 보상을 추첨한다고 오해됨
- 피해자보다 돈·목표가 먼저 변함
- 첫 죽음에 축하가 표시됨
- 초기 룰렛 중 중복 사망·중복 지급이 발생함
- 대상 범위가 명시적 수락 없이 넓어짐
- 효율·가격·잔액이 안전 정수 범위를 벗어남
- 대상군 잔여 또는 `현재 인구 - 1`을 초과해 감소함
- 인구 1 이후 시계·입력·구매가 계속됨
- 저장 복원에서 거래가 사라지거나 두 번 정산됨
- 모바일·키보드·모션 감소 경로가 검수되지 않음
- 사용자 최종 승인이 없음
